## 🚨 High (Высокая серьёзность)

### H-01: Path Traversal через невалидированные пути

**Файл:** `smb_downloader.py`  
**Строки:** ~770 (`_start_copy`), ~960 (`_calc_src_md5`)  
**Серьёзность:** 🟠 **High**

**Описание:**

Пути `src` и `dst` принимаются напрямую из пользовательского ввода без валидации. Приложение формирует путь назначения:

```python
dst_path = os.path.join(dst_dir, os.path.basename(src))
```

Функция `os.path.basename()` на Windows не полностью защищает от path traversal через `..` компоненты.

**Векторы атаки:**
- Ручной ввод пути с `..` в поле источника
- Модификация `history.json` и запуск через контекстное меню
- Редактирование `config.json` с произвольными путями

**Последствия:**
- Перезапись файлов за пределами целевой директории
- Запись в системные директории (при правах администратора)

**Рекомендация:**
```python
# Нормализация путей
src = os.path.normpath(src)
dst_dir = os.path.normpath(dst_dir)

# Для локальных путей — проверка что результат остаётся в пределах dst_dir
dst_path = os.path.normpath(os.path.join(dst_dir, os.path.basename(src)))
```

**Риск снижен:** Пользователь сам вводит пути, программа не принимает данные из сети.

### H-02: Race condition при переименовании после завершения

**Файл:** `smb_downloader.py`  
**Строки:** ~610 (`_on_success`)  
**Серьёзность:** 🟠 **High**

**Описание:**

```python
if os.path.exists(final_path):
    os.remove(final_path)
os.rename(self.part_path, final_path)
```

Между `os.remove()` и `os.rename()` существует временной зазор (TOCTOU). На Windows `os.rename` не работает между разными томами (SMB → локальный диск).

**Последствия:**
- `OSError` при переименовании между разными дисками
- Потеря данных при конфликте имён

**Рекомендация:**
```python
import shutil

# shutil.move работает между томами
shutil.move(self.part_path, final_path)
```

### M-02: Невалидированные данные из `history.json`

**Файл:** `smb_downloader.py`  
**Строки:** ~155 (`_load`), ~945 (`_history_menu_retry`)  
**Серьёзность:** 🟡 **Medium**

**Описание:**

Десериализация `history.json` без проверки структуры. Записи из истории используются для повторного копирования — модифицированный файл может привести к копированию из/в непредусмотренные пути.

**Рекомендация:**
```python
def _load(self):
    if HISTORY_FILE.exists():
        try:
            raw = json.load(f)
            self.entries = []
            required_keys = {"src", "dst", "status"}
            for entry in raw:
                if not isinstance(entry, dict):
                    continue
                if not required_keys.issubset(entry.keys()):
                    continue
                if not isinstance(entry.get("src"), str):
                    continue
                if not isinstance(entry.get("dst"), str):
                    continue
                self.entries.append(entry)
        except Exception:
            self.entries = []
```

### M-03: Symlink-атака на `.part` файлы

**Файл:** `smb_downloader.py`  
**Строки:** ~410 (`_copy_loop`), ~610 (`_on_success`)  
**Серьёзность:** 🟡 **Medium**

**Описание:**

Перед записью в `.part` файл не проверяется, что путь не является символической ссылкой. На Windows это актуально при наличии прав администратора.

**Рекомендация:**
```python
if os.path.islink(self.part_path):
    self._emit_error(f"Целевой .part файл является символической ссылкой")
    return
```

### M-04: `_free_space` не работает с UNC-путями

**Файл:** `smb_downloader.py`  
**Строки:** ~1040 (`_free_space`)  
**Серьёзность:** 🟡 **Medium**

**Описание:**

`shutil.disk_usage()` на Windows не поддерживает UNC-пути (`\\server\share`). Для сетевых ресурсов проверка свободного места всегда возвращает `None`.

**Рекомендация:**
```python
import ctypes

def _free_space(path: str):
    try:
        return shutil.disk_usage(path).free
    except Exception:
        # Для UNC-путей — GetDiskFreeSpaceExW
        try:
            free_bytes = ctypes.c_ulonglong(0)
            ctypes.windll.kernel32.GetDiskFreeSpaceExW(
                ctypes.c_wchar_p(path), None, None, ctypes.byref(free_bytes))
            return free_bytes.value
        except Exception:
            return None
```

## 🔵 Low (Низкая серьёзность)

### L-01: MD5 — криптографически сломанный алгоритм

**Файл:** `smb_downloader.py`  
**Строки:** ~307 (`compute_md5`)  
**Серьёзность:** 🔵 **Low**

**Описание:**

MD5 имеет доказанные коллизии. Для проверки целостности при копировании из доверенных источников это приемлемо, но для ненадёжных источников — недостаточно.

**Риск снижен:** MD5 используется только для **сравнения** двух файлов (источник и приёмник), а не для криптографической аутентификации. Для задачи «файл скопирован без повреждений» MD5 достаточен.

**Рекомендация:** При необходимости добавить SHA-256 как опцию.

**Строки:** ~1425 (`_history_menu_open_folder`)  
**Серьёзность:** 🔵 **Low**

os.startfile(dst_dir)

### L-03: Неатомарная запись `config.json`

**Файл:** `smb_downloader.py`  
**Строки:** ~142 (`save_config`)  
**Серьёзность:** 🔵 **Low**

**Описание:**

При сбое во время записи `config.json` может оказаться повреждённым. Приложение корректно обрабатывает это (fallback на defaults), но настройки будут потеряны.

**Рекомендация:**
```python
def save_config(cfg: dict):
    tmp = CONFIG_FILE.with_suffix(".tmp")
    with open(tmp, "w", encoding="utf-8") as f:
        json.dump(cfg, f, indent=2)
    os.replace(tmp, CONFIG_FILE)  # атомарная замена
```

### L-04: Нет ограничения на размер `.meta` файла

**Файл:** `smb_downloader.py`  
**Строки:** ~232 (`load_meta`)  
**Серьёзность:** 🔵 **Low**

**Описание:**

`json.load(f)` читает весь файл в память. Подменённый `.meta` файл большого размера может вызвать DoS.
