# Changelog

Все значительные изменения в этом проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.0.0/).

---

## [3.0.0] — 2026-04-07

### Исправления
- **Критическое:** Исправлена пауза — поток корректно завершается, метаданные сохраняются, кнопка «Старт» активируется для возобновления
- **Критическое:** MD5 больше не проверяется при паузе (только при полном завершении)
- Исправлен дублирующий вызов `lbl_status.config()` при возобновлении (`text=` kwarg)
- Убрано блокирующее `pause_event.wait()` — пауза теперь завершает поток
- Добавлено ожидание предыдущего потока при повторном нажатии «Старт»
- Корректная очистка `.part`/`.meta` при отмене копирования

### Улучшения
- Пауза → «Старт» → возобновление с места остановки (полный цикл проверен тестом)
- Автоопределение незавершённых `.part` файлов при запуске
- Двойной клик по записи истории «в процессе» — предложение докачать
- Кнопка «О программе» с описанием возможностей

## [2.1] — 2026-04-07

### Изменения
- Удалена очередь загрузок (упрощение интерфейса)
- Удалён `QueueManager` и вкладка «Очередь»
- Удалены зависимости: `threading.Lock`
- Интерфейс сведён к двум вкладкам: «Копирование» и «История»

## [2.0] — 2026-04-07

### Новое
- **Динамический размер блока** — автоподстройка от 32 КБ до 8 МБ на основе скорости
- **Умная пауза при потере сети** — экспоненциальная задержка (5с → 10с → 20с → … → 5мин), до 10 попыток
- **Автосохранение прогресса** — каждые 30 секунд или каждые 10 МБ (что раньше)
- **История копирований** — Treeview с 8 колонками, экспорт в CSV, до 50 записей
- **Визуальный индикатор скорости** — цветной Progressbar (зелёный/жёлтый/красный)
- **Настройки в config.json** — скорость, папка, режим блока, retry, геометрия окна
- **Кнопка «Повторить (сеть)»** — ручной перезапуск после потери сети

## [1.0] — 2026-04-06

### Первое издание
- Копирование файлов с сетевых дисков (SMB)
- Возобновление через `.part` + `.meta` файлы
- Ограничение скорости (KB/s)
- Прогресс-бар, ETA, отображение скорости
- Проверка MD5 после завершения
- Логирование в `copy_log.txt`
- Кнопки: Старт / Пауза / Отмена
- Обработка ошибок: нет доступа, нет места, обрыв связи
- Сборка в .exe через PyInstaller
