# Changelog

Все значимые изменения в проекте SMB-Downloader.

Формат соответствует [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версионирование по [Semantic Versioning](https://semver.org/lang/ru/).

---

## [3.1.0] — 2026-04-08

### Добавлено
- Кнопка **🔒 MD5** для предварительного расчёта хэша исходного файла до начала копирования
- Раздельные записи в истории: отдельная запись для расчёта MD5, отдельная — для копирования
- Статус **«MD5 рассчитан»** в истории для записей предварительного хэша
- Две колонки MD5 в истории: **«MD5 источника»** и **«MD5 приёмника»**
- **Двойной клик** по записи в истории — открывает папку с файлом в проводнике (с выделением файла)
- **Контекстное меню** истории (правый клик):
  - «Повторить копирование»
  - «Открыть папку с файлом»
- Кнопка **«Отмена»** теперь очищает поля ввода (исходный файл и папка назначения)
- Кнопка **«Отмена»** предупреждает о потере прогресса перед подтверждением
- Сброс кнопки MD5 при очистке полей
- Предупреждение о потере прогресса при отмене копирования

### Изменено
- История копирования теперь создаёт **новую записи** при каждом запуске (не обновляет существующую)
- Функция `find_in_progress()` ищет записи со статусами: «в процессе», «MD5 рассчитан», «пауза»
- Колонки истории: `md5_match` заменены на `md5_src` и `md5_dst`
- Экспорт CSV обновлён для новых колонок MD5
- Окно «О программе» дополнено новыми возможностями v3.1
- Версия приложения обновлена до 3.1

## [3.0.0] — 2026-04-07

### Добавлено
- **Возобновление копирования** после обрыва связи или выключения ПК (файлы `.part` / `.meta`)
- **Ограничение скорости** — настраиваемый лимит в КБ/с
- **Прогресс-бар** с отображением:
  - Процента выполнения
  - Скопированного объёма
  - Текущей скорости
  - ETA (оставшееся время)
- **Визуальный индикатор скорости** — цветная полоса (зелёный → жёлтый → красный)
- **MD5-верификация** — автоматическая проверка целостности после копирования
- **Динамический размер блока** (#8) — автоподстройка от 32 КБ до 8 МБ
- **Экспоненциальная задержка** при потере сети (#5) — retry с нарастающим интервалом
- **Автосохранение прогресса** (#11) — каждые 30 секунд / 10 МБ
- **История копирования** (#13):
  - Вкладка «История» с таблицей всех операций
  - Экспорт в CSV
  - Очистка истории
  - Двойной клик по записи «в процессе» — предложение докачать
- **Автоопределение незавершённых копиований** при запуске — поиск `.part`/`.meta` файлов
- **Конфигурация** (`config.json`) — сохранение настроек между запусками
- **Логирование** всех событий в `copy_log.txt`
- **Проверка свободного места** перед началом копирования
- **Возобновление после потери сети** — кнопка «🔄 Повторить (сеть)»
- **Всплывающие уведомления** — статус копирования в реальном времени
- **Поддержка UNC-путей** — работа с `\\server\share\...`

### Исправлено
- **[FIX]** Пауза/возобновление — корректное переоткрытие файлов
- **[FIX]** MD5 проверяется ТОЛЬКО при полном завершении (не при паузе/отмене)
- **[FIX]** Запись в историю при старте (статус «в процессе»)
- **[FIX]** После потери сети — переоткрытие файлов перед продолжением
- **[FIX]** Сохранение метаданных при закрытии окна

### Технические детали
- Многопоточность — копирование в отдельном потоке, UI не блокируется
- Event-based пауза/отмена через `threading.Event`
- Формат метаданных: JSON с полями source, destination, total_size, copied_bytes, interruptions
- Все callback'и из потока в UI через `root.after()`
