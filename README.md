## Возможности

| Функция | Описание |
|---|---|
| **Возобновление копирования** | После обрыва связи или выключения ПК копирование продолжается с места останова |
| **Предварительный расчёт MD5** | Рассчитайте хэш источника до начала копирования для последующего сравнения |
| **Ограничение скорости** | Настраиваемый лимит в КБ/с для снижения нагрузки на сеть (0 = без ограничений) |
| **Прогресс-бар** | Отображение процента, скопированного объёма, текущей скорости и оставшегося времени (ETA) |
| **MD5-верификация** | Автоматическая проверка целостности файла после копирования |
| **Раздельные записи в истории** | Отдельные записи для расчёта MD5 и для копирования |
| **Динамический размер блока** | Автоматическая подстройка размера чанка (32 КБ – 8 МБ) в зависимости от скорости |
| **Экспоненциальная задержка** | Умный retry при потере сети с нарастающим интервалом |
| **Автосохранение прогресса** | Каждые 30 секунд / 10 МБ для безопасности данных |
| **История с экспортом в CSV** | Все операции записываются, можно выгрузить в таблицу |
| **Контекстное меню истории** | Правый клик — повторить копирование или открыть папку с файлом |
| **Пауза / Отмена / Старт** | Полный контроль над процессом копирования |
| **Логирование** | Все события записываются в `copy_log.txt` с временными метками |
| **Файлы любого размера** | Корректная работа с файлами 4 ГБ и более |
| **UNC-пути** | Полная поддержка сетевых путей вида `\\server\share\file.ext` |

## Что нового в v3.1

### 🆕 Предварительный расчёт MD5

Новая кнопка **🔒 MD5** рядом с полем исходного файла позволяет рассчитать хэш файла **до** начала копирования. Результат записывается в историю как отдельная запись со статусом «MD5 рассчитан».

**Зачем это нужно:**
- Вы видите хэш источника **до** копирования
- После завершения в истории отображаются **оба хэша** — источника и приёмника
- Можно визуально сравнить и убедиться, что файл скопирован без повреждений

### 📋 Раздельные записи в истории

Раньше расчёт MD5 и копирование обновляли одну запись. Теперь — **две отдельные записи**:
1. Первая — с результатом предварительного расчёта MD5
2. Вторая — с результатами копирования (скорость, прерывания, оба хэша)

### 🖱 Двойной клик и контекстное меню в истории

| Действие | Результат |
|---|---|
| **Двойной клик** по записи | Открывает папку с файлом в проводнике (с выделением файла) |
| **Правый клик** | Контекстное меню: «Повторить копирование» / «Открыть папку с файлом» |

### 🛑 Обновлённая кнопка «Отмена»

Теперь при отмене копирования:
- Временные файлы (`.part`, `.meta`) удаляются
- **Поля ввода очищаются** — готовность к новой операции
- В истории фиксируется статус «отмена»
- Появляется предупреждение о потере прогресса

### ✨ Обновлённый интерфейс «О программе»

Добавлены новые возможности v3.1 в описание.

## Использование

### Пошаговое руководство

1. **Выберите исходный файл**
   - Нажмите кнопку **Обзор…** в секции «Исходный файл».
   - В диалоге перейдите к нужному файлу (локальному или сетевому).
   - Или вставьте UNC-путь вручную: `\\server\share\folder\file.iso`.

2. **Рассчитайте MD5 (необязательно, но рекомендуется)**
   - Нажмите кнопку **🔒 MD5** рядом с полем файла.
   - Дождитесь завершения расчёта — прогресс отображается в статусе.
   - Результат автоматически запишется в историю.

3. **Выберите папку назначения**
   - Нажмите **Обзор…** в секции «Папка назначения».
   - Укажите локальную папку для сохранения файла.

4. **Настройте параметры (необязательно)**
   - **Лимит скорости** — введите значение в КБ/с (0 = без ограничений).
   - **Размер блока** — «Авто» (рекомендуется) или «Ручной».
   - **Попытки** — количество и базовый интервал повторных попыток при потере сети.

5. **Начните копирование**
   - Нажмите **▶ Старт**.
   - Прогресс-бар и статус начнут обновляться в реальном времени.

6. **Пауза и возобновление**
   - Нажмите **⏸ Пауза** для временной остановки.
   - Для возобновления снова нажмите **▶ Старт** — копирование продолжится с места останова.

7. **Отмена**
   - Нажмите **⏹ Отмена** и подтвердите действие.
   - Прогресс будет потерян, поля ввода очищены.

8. **Завершение**
   - После копирования автоматически выполняется проверка MD5.
   - В истории появятся обе хэш-суммы для сравнения.
   - Временные файлы `.part` и `.meta` удаляются.

### Работа с историей

| Действие | Описание |
|---|---|
| **Двойной клик** по записи | Открывает папку с файлом в проводнике |
| **Правый клик** | Контекстное меню: повторить копирование / открыть папку |
| **📁 Экспорт CSV** | Сохранить всю историю в CSV-файл |
| **🗑 Очистить** | Удалить все записи из истории |

### Записи «в процессе» и «пауза»

Двойной клик по такой записи предложит **возобновить копирование** с сохранённой позиции (если файлы `.part`/`.meta` существуют).

### Логирование

Все события записываются в файл `copy_log.txt`, который создаётся в той же папке, что и скрипт.

Пример записи:

```
2026-04-08 10:15:30 [INFO] Start: \\server\share\movie.mkv -> C:\Users\Seleb\Downloads\movie.mkv.part (3,89 ГБ)
2026-04-08 10:28:45 [INFO] Copy finished. MD5 check…
2026-04-08 10:29:12 [INFO] MD5 match: a3f1c8e9d2b7f014e56c7890abcdef12
```

## Сборка .exe через PyInstaller

### Предварительная подготовка

```cmd
pip install pyinstaller
```

### Команда сборки

Перейдите в папку с проектом и выполните:

```cmd
pyinstaller --clean build.spec
```

Или вручную:

```cmd
pyinstaller --onefile --windowed --name "SMB-Downloader-v3.1" smb_downloader.py
```

| Флаг | Назначение |
|---|---|
| `--onefile` | Упаковать всё в один `.exe` |
| `--windowed` | Не открывать консольное окно (GUI-режим) |
| `--name` | Имя выходного файла |

### Результат

После завершения сборки `.exe` находится в папке:

```
dist\SMB-Downloader-v3.1.exe
```

## Технические детали

### Как работает возобновление

Механизм возобновления основан на вспомогательных файлах:

#### Файл `.part` (частично скопированные данные)

- Временный файл, в который записываются данные блоками.
- При успешном завершении переименовывается в финальный файл.
- Если копирование прервано — `.part` остаётся на диске.

#### Файл `.meta` (метаданные)

JSON-файл, хранящий состояние копирования:

```json
{
  "source": "\\\\server\\share\\movie.mkv",
  "destination": "C:\\Users\\Seleb\\Downloads\\movie.mkv",
  "total_size": 4176842752,
  "copied_bytes": 2310456320,
  "timestamp": "2026-04-08T10:15:30.123456",
  "interruptions": 2
}
```

#### Файл `.part.meta` (автосохранение)

Создаётся каждые 30 секунд или каждые 10 МБ скопированных данных для дополнительной защиты.

#### Алгоритм возобновления

```
1. При нажатии «Старт» проверяется наличие .meta-файла.
2. Если .meta существует → извлекается copied_bytes.
3. Если .part существует и copied_bytes < total_size:
   → Копирование продолжается с места останова.
4. Если .meta нет или .part удалён:
   → Копирование начинается заново.
```

### Предварительный расчёт MD5

Кнопка **🔒 MD5** запускает расчёт хэша в отдельном потоке, чтобы не замораживать интерфейс. Создаётся отдельная запись в истории:

```json
{
  "src": "\\\\server\\share\\movie.mkv",
  "dst": "C:\\Users\\Seleb\\Downloads\\movie.mkv",
  "size": "3,89 ГБ",
  "datetime": "2026-04-08 10:10",
  "status": "MD5 рассчитан",
  "md5_src": "a3f1c8e9d2b7f014e56c7890abcdef12",
  "md5_dst": "—"
}
```

После копирования — вторая запись с заполненными полями `md5_src` и `md5_dst`.

### Проверка MD5 после копирования

1. Вычисляется MD5 исходного файла (чтение чанками по 8 МБ).
2. Вычисляется MD5 скопированного файла.
3. Хэши сравниваются:
   - **Совпали** → успех, временные файлы удалены.
   - **Не совпали** → ошибка с выводом обоих хэшей.

### Динамический размер блока (#8)

Автоматическая подстройка:

| Средняя скорость | Целевой размер блока |
|---|---|
| > 10 МБ/с | 4 МБ |
| > 5 МБ/с | 1 МБ |
| < 100 КБ/с | 64 КБ |
| < 500 КБ/с | 128 КБ |
| Остальное | 256 КБ (старт) |

Размер блока меняется плавно, не более чем в 2 раза за итерацию.

### Экспоненциальная задержка при потере сети (#5)

При обрыве соединения:

```
Попытка 1: ждать 5 с
Попытка 2: ждать 10 с
Попытка 3: ждать 20 с
...
Попытка N: ждать min(5 * 2^(N-1), 300) с
```

Максимум 10 попыток, максимум 300 секунд между ними.

## Устранение неполадок

### Ошибка: «Исходный файл не найден»

**Решение:**
1. Проверьте доступ к сетевой папке через Проводник.
2. Убедитесь, что UNC-путь указан верно: `\\server\share\file`.
3. При необходимости подключите сетевой диск:

   ```cmd
   net use Z: \\server\share /user:DOMAIN\username password
   ```

### Ошибка: «Недостаточно места на диске»

**Решение:**
- Освободите место на целевом диске или выберите другой диск.

### MD5 не совпадает

**Причины:**
- Повреждение данных при передаче по сети.
- Файл был изменён во время копирования.

**Решение:**
1. Проверьте стабильность сетевого соединения.
2. Попробуйте скопировать файл заново.

### Ошибка доступа (PermissionError)

**Решение:**
- Убедитесь, что у вас есть права на чтение исходного файла.
- Запустите приложение от имени администратора (если требуется).

### Копирование зависло

**Решение:**
- Нажмите **⏸ Пауза**, затем **▶ Старт** для перезапуска.
- При потере сети — кнопка **🔄 Повторить (сеть)** станет активной.
