## Возможности

- **Возобновление копирования** — после обрыва связи или выключения ПК (`.part` + `.meta`)
- **Ограничение скорости** — настройка в КБ/с для слабых каналов
- **Динамический размер блока** — автоподстройка от 32 КБ до 8 МБ
- **Умная обработка потери сети** — экспоненциальная задержка, до 10 попыток
- **Автосохранение прогресса** — каждые 30 секунд / 10 МБ
- **Прогресс-бар, ETA, скорость** — обновление в реальном времени
- **Визуальный индикатор скорости** — цветной прогресс-бар (зелёный → жёлтый → красный)
- **Проверка целостности MD5** — после каждого завершённого копирования
- **История копирований** — с экспортом в CSV
- **Автообнаружение незавершённых файлов** — при запуске предлагает продолжить

## Установка

### Из исходного кода

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/selebren/smb-downloader.git
cd smb-downloader

# 2. Запустите
python smb_downloader.py
```

### Готовый .exe

Скачайте `SMB-Downloader-v3.exe` из [Releases](../../releases) и запустите. Установка Python не требуется.

## Использование

1. Нажмите **«Обзор…»** и выберите файл на сетевом диске
2. Выберите **папку назначения** на локальном компьютере
3. При необходимости задайте **лимит скорости** (0 = без ограничений)
4. Нажмите **«▶ Старт»**

### Пауза и возобновление

| Действие | Описание |
|---|---|
| **⏸ Пауза** | Остановить копирование с сохранением позиции. Кнопка «Старт» станет активной |
| **▶ Старт** (после паузы) | Возобновить копирование с места остановки |
| **⏹ Отмена** | Остановить и удалить временные файлы (`.part`, `.meta`) |
| **🔄 Повторить (сеть)** | Перезапустить после потери сети |

### При потере сети

Программа автоматически:
1. Ждёт с экспоненциальной задержкой: 5с → 10с → 20с → 40с → … → 5мин
2. Повторяет до 10 попыток (настраивается)
3. После восстановления — продолжает с того же места
4. Если все попытки исчерпаны — кнопка «Повторить (сеть)»

## Сборка в .exe

```bash
pip install pyinstaller
pyinstaller --onefile --windowed --name "SMB-Downloader-v3" \
  --version-file version.txt \
```

### Файлы данных

| Файл | Назначение |
|---|---|
| `config.json` | Настройки программы (скорость, блок, retry) |
| `history.json` | История копирований (до 50 записей) |
| `copy_log.txt` | Журнал событий |
| `<file>.part` | Временный файл копирования |
| `<file>.meta` | Метаданные для возобновления (позиция, размер) |
| `<file>.part.meta` | Автосохранение прогресса |

Все файлы хранятся в **той же папке**, что и исполняемый файл (или `.py` скрипт).

## Устранение неполадок

| Проблема | Решение |
|---|---|
| Не найден исходный файл | Проверьте путь и доступ к сетевому диску |
| Недостаточно места | Освободите место на целевом диске |
| Копирование не возобновляется | Убедитесь что `.part` и `.meta` файлы не удалены |
| MD5 не совпадает | Файл повреждён — попробуйте снова (`.part` сохранён) |
| «Сеть недоступна» | Проверьте подключение, нажмите «Повторить (сеть)» |
| Окно не открывается | Проверьте установку Python с опцией tcl/tk |
